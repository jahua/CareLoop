### Writing Expectations (HSLU-aligned)

- **General principles**
  - Write clearly and concisely; define terms on first use; avoid redundancy.
  - Maintain formal academic tone; prefer active voice where appropriate.
  - Use gender-inclusive, respectful language; be consistent (British or American English).

- **Document structure (order)**
  - Title page (program, title, author, date, supervisors, HSLU identifiers as required).
  - Declaration of originality (signed, dated) and AI-tools statement if required.
  - Abstract (concise: problem, method, key results, implications).
  - Acknowledgements (optional).
  - Table of contents.
  - List of figures; list of tables; list of abbreviations and symbols (if used).
  - Main body (chapters with numbered headings).
  - References (complete and consistent style).
  - Appendices (prompts, configs, evaluation matrices, extended tables/figures).

- **Formatting and typography**
  - Page: A4, standard margins; ensure sufficient white space for binding and readability.
  - Font: readable serif or sans-serif; consistent sizes (e.g., 11–12 pt body); avoid mixing fonts.
  - Line spacing: 1.15–1.5 for body text; single spacing for captions, footnotes, references.
  - Paragraphs: consistent spacing; avoid orphan/widow lines; enable hyphenation if justified.
  - Alignment: left or justified; avoid overly stretched spacing.
  - Page numbers: front matter often roman (i, ii, iii); main body Arabic (1, 2, 3) starting at Introduction.

- **Headings and numbering**
  - Number headings hierarchically (1, 1.1, 1.1.1); keep to ≤3–4 levels in TOC.
  - Ensure heading hierarchy is consistent and informative; avoid single subheadings.
  - Cross-reference sections using automatic references (avoid hard-coded numbers).

- **Figures, tables, and equations**
  - Number by chapter (e.g., Figure 2.3, Table 3.1); use consistent caption style.
  - Placement: table captions above; figure captions below; include source notes when applicable.
  - Reference all figures/tables in the text; ensure readability (sufficient resolution/contrast).
  - Equations: center or align consistently; number if referenced (e.g., (2.1)).

- **Citations and references**
  - Choose one style (APA 7th or IEEE) and use it consistently throughout.
  - In-text citations and reference list must match exactly; include DOIs/URLs where applicable.
  - Prefer primary, peer-reviewed sources; avoid uncited claims.
  - Use a reference manager to prevent inconsistencies; validate against style guide.

- **Language and style**
  - Use precise, evidence-based statements; qualify uncertainty.
  - Tense: past for completed methods/results; present for general facts; future only for planned work.
  - Avoid colloquialisms, rhetorical questions, and unsupported normative claims.

- **Ethics, data protection, consent**
  - Describe data handling, storage location, retention period, and access control.
  - Anonymize personal data; collect informed consent for any human data.
  - Include an ethics note or approval statement if applicable to your program.

- **Reproducibility and artifacts**
  - Record tool versions (LLM models, libraries), prompts, seeds, configs.
  - Provide paths or repositories for datasets and code; document preprocessing steps.
  - Log evaluation procedures and metrics; include configuration files in the appendix.

- **Accessibility and usability**
  - Provide alt-text for figures where feasible; avoid color-only encodings; ensure contrast.
  - Keep table structures simple; avoid images of text for core content.

- **Pre-submission checklist**
  - Update TOC, lists of figures/tables; check cross-references (no “Error”/“??”).
  - Validate citations: no missing entries; consistent style; all URLs/DOIs working.
  - Spellcheck and grammar pass; unify English variant; consistent terminology.
  - Verify figure/table numbering, captions, and sources; ensure print legibility.
  - Confirm pagination scheme (front matter vs main body) and header/footer consistency.
  - Export submission PDF (PDF/A if required); test on multiple viewers and printers.
