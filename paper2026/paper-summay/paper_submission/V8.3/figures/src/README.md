# Figure Generation Scripts

This directory contains all source code for generating manuscript diagrams.

## Directory Structure

```
src/
├── README.md                          # This file
├── generate_all.py                    # Master script - run this to regenerate all figures
├── diagram_theme.py                   # Shared styling & utilities
├── create_system_overview.py          # Generates Figure 10 (system overview)
├── create_submission_diagrams.py      # Generates Figures 12, 13 (evaluation, pipeline)
├── create_study_design_flowchart.py   # Generates Figure 11 (study design)
├── create_detection_architecture.py   # Generates Figure 14 (layered detection pipeline)
├── 14_detection_pipeline.dot          # Graphviz source (alternative Figure 14)
├── 15_regulation_prompt_assembly.dot  # Graphviz source for Figure 15
├── 16_trait_to_zurich_mapping.dot     # Graphviz source for Figure 16
└── requirements.txt                   # Python dependencies
```

## Quick Start

To regenerate all diagrams:

```bash
cd figures/src
python generate_all.py
```

Output will be saved to parent `figures/` directory.

## Requirements

- Python 3.9+
- matplotlib

Install dependencies:
```bash
pip install matplotlib
```

## Script Details

### `create_system_overview.py`

Generates the main system overview figure (Figure 10) showing:
- User Input & Context → Personality Detection → Behavior Adaptation → Response (main flow)
- Quality Assurance layer: AI Evaluation, Human Experts, Statistical Analysis
- Key Results summary with effect sizes
- Clean, simple topology matching manuscript Section 3.2

### `create_detection_architecture.py`

Generates the layered detection pipeline figure (Figure 14 - recommended version) showing:
- **Input Layer**: User message + dialogue context (top)
- **Inference Layer**: Five parallel OCEAN trait detectors running simultaneously (middle)
- **State & Interface Layer**: Personality state update with cumulative evidence + interface to regulation module (bottom)
- Clear left-to-right data flow, no crossing arrows
- Emphasizes discrete personality estimates with confidence scores
- Shows τ = 0.7 confidence threshold for state transitions

This architecture makes it immediately clear to reviewers that:
- Trait detection is modular and parallelized
- The system accumulates evidence over multiple turns
- Output is an explicit interface (not an action)

### `diagram_theme.py`

Shared utilities for consistent figure styling:
- Color palette (blue, green, gray, lavender)
- Typography settings (Inter/Source Sans Pro/Helvetica)
- Box, arrow, and text drawing functions
- Canvas setup and save functions

### `create_submission_diagrams.py`

Generates multiple conceptual diagrams:
- `fig02_system_architecture()` - 5-layer architecture (input → detection → regulation → generation → evaluation)
- `fig03_data_flow()` - Data processing pipeline
- `fig04_detection_process()` - Detection system details
- `fig05_theoretical_framework()` - Theoretical foundations
- `fig06_regulation_system()` - Regulation mechanism
- `fig07_evaluation_framework()` - Evaluation criteria & process

### `create_study_design_flowchart.py`

Generates the main study design flowchart showing:
- Study setup (20 agents, 2 personality types)
- Personality profiles (Type A high-functioning, Type B vulnerable)
- Experimental conditions (regulated vs baseline)
- Dialogue generation (120 turns)
- Evaluation (AI + human validation)
- Statistical analysis & key findings

## Customization

All diagrams share the same visual system:
- **Colors**: Muted, print-friendly palette (blue for input/detection, green for regulation, gray for processing, lavender for evaluation/analysis)
- **Typography**: Sans-serif fonts (Inter → Source Sans Pro → Helvetica → Arial fallback)
- **Shapes**: Rounded rectangles with subtle borders
- **Arrows**: Bold, grayscale-safe directional indicators
- **Spacing**: Consistent padding and alignment grid

To modify styling, edit `diagram_theme.py` Theme class colors.

To modify diagram content/layout, edit the specific `fig*()` functions in the generator scripts.

## Output Format

- **Format**: PNG
- **DPI**: 300 (publication quality)
- **Background**: White
- **Dimensions**: Variable (optimized per diagram)

## Troubleshooting

### Missing fonts

If you see font warnings, the scripts will automatically fall back to system fonts. For best results, install Inter or Source Sans Pro:
- Inter: https://rsms.me/inter/
- Source Sans Pro: https://adobe-fonts.github.io/source-sans/

### Import errors

Make sure you're running from the `src/` directory:
```bash
cd figures/src
python generate_all.py
```

### Output directory

All figures are saved to `figures/` (parent directory). The `OUT` variable in each script points to `Path(__file__).resolve().parents[1]` (parent of src/).

## Maintenance

When adding new figures:
1. Add generation function to appropriate script (or create new script)
2. Update `generate_all.py` to include new script
3. Update this README with figure details
4. Test generation: `python generate_all.py`

## Version Control

These scripts are self-contained in `figures/src/`. All dependencies are explicit:
- Python stdlib only (pathlib, subprocess)
- matplotlib (specified in requirements)
- No external data files needed

For archival/reproducibility, this directory can be distributed standalone with Python + matplotlib.
