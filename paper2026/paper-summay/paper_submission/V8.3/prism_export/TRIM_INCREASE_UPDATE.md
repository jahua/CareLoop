### 根本原因

**matplotlib 生成的高分辨率图片（600 DPI）包含大量白边**：

```python
# 在生成脚本中
matplotlib.rcParams['figure.dpi'] = 600
fig, ax = plt.subplots(figsize=(14, 12), dpi=600)
```

**问题**:
- `figsize=(14, 12)` 创建了一个很大的画布
- 即使使用 `bbox_inches='tight'`，matplotlib 仍会保留一些边距
- 600 DPI 意味着这些边距在 LaTeX 中会显得很大
- **之前的 20pt 裁剪远远不够**

## 🎨 参数详解

### LaTeX trim 语法

```latex
\includegraphics[trim=left bottom right top,clip]{file}
```

**单位**: pt (points)  
**我们的设置**: `trim=0 80 0 80`

| 边 | 值 | 说明 |
|----|---|------|
| **left** | 0pt | 左边不裁剪 |
| **bottom** | 80pt | 底部裁剪 80pt ⭐ |
| **right** | 0pt | 右边不裁剪 |
| **top** | 80pt | 顶部裁剪 80pt ⭐ |

### 为什么是 80pt？

**估算依据**:
1. MDPI 图片尺寸: ~7000 x 4000 像素 @ 600 DPI
2. LaTeX 页面宽度: ~6.5 inches = 468pt
3. 图片缩放后高度: ~260pt (估算)
4. 原始白边: ~50-60pt (视觉估算)
5. **需要裁剪**: 40-50pt 才能减少到 ~10pt

**80pt 是一个保守的值**，确保：
- ✅ 去除大部分白边
- ✅ 不会裁剪到实际内容
- ✅ 与统计图表的间距一致
